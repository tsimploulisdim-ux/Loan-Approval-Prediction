# Loan-Approval-Prediction
Μοντελοποίηση Μηχανικής Μάθησης για την Πρόβλεψη και Αξιολόγηση Πιστωτικού Κινδύνου Ιδιωτών 

1. Επιχειρηματική Αξία της Λύσης (Business Value) 

Στον σύγχρονο χρηματοπιστωτικό τομέα, η χειροκίνητη αξιολόγηση των αιτήσεων δανεισμού είναι μια διαδικασία χρονοβόρα, υποκειμενική και επιρρεπής σε ανθρώπινα σφάλματα. Η παρούσα λύση προσφέρει άμεση επιχειρηματική αξία σε έναν οργανισμό μέσω: 

Μείωσης του Λειτουργικού Κόστους: Αυτοματοποίηση της διαδικασίας έγκρισης/απόρριψης για τις προφανείς περιπτώσεις, επιτρέποντας στους αναλυτές να εστιάσουν σε σύνθετες αιτήσεις. 

Μετριασμού του Ρίσκου (Risk Mitigation): Έγκαιρος εντοπισμός "κόκκινων" δανείων (False Positives) πριν την εκταμίευση, προστατεύοντας τα κεφάλαια της τράπεζας. 

Βελτίωσης της Εμπειρίας Πελάτη (Customer Experience): Ταχύτερη έκδοση αποφάσεων, γεγονός που αυξάνει την ανταγωνιστικότητα του οργανισμού στην αγορά. 

2. Αρχιτεκτονική Προσέγγιση & Pipeline 

Το project υλοποιήθηκε με τη χρήση της βιβλιοθήκης scikit-learn ακολουθώντας την αρχή του Modular Code (Αρθρωτός Κώδικας) για εύκολη παραγωγή (production-ready deployment): 

Α. Μηχανική Χαρακτηριστικών (Feature Engineering) 

Συντέθηκαν δύο εξειδικευμένα χαρακτηριστικά: 

TotalIncome: Το συνολικό διαθέσιμο εισόδημα του νοικοκυριού. 

DebtIncomeRatio: Ο δείκτης μόχλευσης (αιτούμενο ποσό / εισόδημα), ο οποίος αποτελεί τον πυρήνα της πιστοληπτικής αξιολόγησης. 

Β. Data Pipeline (ColumnTransformer) 

Διαχωρίσαμε τις μεταβλητές για να αποφευχθεί το Data Leakage: 

Αριθμητικές: SimpleImputer(median) για σταθερότητα έναντι των outliers και StandardScaler για ομογενοποίηση των κλιμάκων. 

Κατηγορικές: SimpleImputer(most_frequent) και OneHotEncoder(handle_unknown="ignore") για ασφαλή διαχείριση νέων, άγνωστων κατηγοριών κατά το testing. 

3. Σύγκριση Μοντέλων & Μεθοδολογία Αξιολόγησης 

Εκπαιδεύτηκαν τρία μοντέλα διαφορετικής πολυπλοκότητας για τη διασφάλιση της αρτιότητας της προσέγγισης: 

Logistic Regression: Ως ερμηνεύσιμο baseline μοντέλο. 

Random Forest Classifier: Με παραμετροποίηση class_weight="balanced" για τη διαχείριση της ανισορροπίας των κλάσεων (Class Imbalance). 

Gradient Boosting Classifier: Για τη μεγιστοποίηση της γενικευτικής ικανότητας.Η αξιολόγηση πραγματοποιείται με βάση το AUC-ROC Score και το F1-Score, καθώς η απλή Ακρίβεια (Accuracy) είναι παραπλανητική σε δεδομένα με ανισόρροπες κλάσεις. 

4. Διαφάνεια, Παρόμοιες Προσεγγίσεις & Αναφορά Πηγών 

Πηγή Δεδομένων & Βάση Κώδικα: Το dataset που χρησιμοποιήθηκε βασίζεται στο κλασικό "Loan Prediction Dataset" (διαθέσιμο σε πλατφόρμες όπως το Kaggle και το GitHub).  

Διαφοροποίηση της λύσης: Ενώ οι περισσότερες δημόσιες προσεγγίσεις περιορίζονται σε απλή εκτέλεση blocks κώδικα σε Jupyter Notebooks, η δική μας προσέγγιση διαφοροποιείται διότι: 

Ενσωματώνει πλήρη Pipelines που επιτρέπουν στο μοντέλο να δεχτεί ακατέργαστα (raw) δεδομένα χωρίς να κρασάρει. 

Εξάγει αυτοματοποιημένα δομημένα reports (results.csv) και visual διαγράμματα σε ανεξάρτητο φάκελο (images/), έτοιμα για ενσωμάτωση σε business dashboards. 

5. Αναγνώριση Αδυναμιών & Μελλοντικές Βελτιώσεις (Limitations) 

Κάθε τεχνικό project οφείλει να αναγνωρίζει τα όριά του:  

Έλλειψη Χρονικής Διάστασης: Τα δεδομένα είναι στατικά. Σε πραγματικές συνθήκες, η συμπεριφορά των δανειοληπτών επηρεάζεται από μακροοικονομικούς παράγοντες (π.χ. πληθωρισμός, αλλαγή επιτοκίων). 

Ερμηνευσιμότητα (Black-Box): Αν και το Random Forest δίνει Feature Importances, για τραπεζικά δεδομένα θα ήταν προτιμότερη η ενσωμάτωση βιβλιοθηκών όπως το SHAP ή το LIME για την ακριβή επεξήγηση κάθε μεμονωμένης απόρριψης δανείου (Explainable AI - XAI). 

Hyperparameter Tuning: Στο παρόν στάδιο οι παράμετροι ορίστηκαν ευριστικά. Μια μελλοντική βελτίωση περιλαμβάνει την προσθήκη GridSearchCV ή RandomizedSearchCV για την εύρεση των βέλτιστων υπερπαραμέτρων
